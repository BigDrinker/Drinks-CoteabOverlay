# Bug fixes applied

## 1. Native overlay closes with the macro

- `Api.close_window()` now calls `hide_overlay()` before destroying the main window.
- The `main()` shutdown `finally` block also calls `api.hide_overlay()`.
- This covers both the app's Close button and the normal Windows title-bar X.

## 2. Discord session runtime is accurate

The old code increased `current_session` by one every time the UI requested the runtime. Because the UI refreshes many times per second, a roughly five-hour session could be reported as around 48 hours.

- `current_session` now stores elapsed seconds from `start_time`.
- On stop, the exact current-run duration is frozen before the Discord summary is sent.

## 3. Duplicate biome Discord notifications are suppressed

- Duplicate webhook URLs are normalized and removed while preserving their original order.
- Identical biome/event notifications are suppressed for eight seconds.
- Start and end notifications are tracked separately, so a legitimate biome end is not blocked by its start notification.

## Validation performed

All Python files were compiled with `python -m compileall` after the edits. This confirms syntax/import-time compilation, but the macro should still be tested from source before creating a shareable EXE.
